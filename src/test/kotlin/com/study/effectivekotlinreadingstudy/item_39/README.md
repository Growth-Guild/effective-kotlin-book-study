# Item 39. 태그 클래스보다 클래스 계층을 사용하라

- - -

* 큰 규모의 프로젝트에서는 상수 '모드'를 가진 클래스를 많이 볼 수 있는데, 이러한 상수 모드를 태그 클래스라고 한다.
* 태그 클래스는 서로 다른 책임을 한 클래스에 태그로 구분해서 넣는다는 점이다.

### 태그 클래스의 단점
* 한 클래스에 여러 모드를 처리하기 위한 상용구가 추가된다.
* 요소가 여러 목적으로 사용해야 하므로 프로퍼티가 일관적이지 않게 사용될 수 있다.
* 팩토리 메서드를 사용해야 하는 경우가 많다.

### sealed class
* 한 클래스에 여러 모드를 만드는 방법 대신에, 각각의 모드를 여러 클래스로 만들고 타입 시스템과 다형성을 활용한다.
* sealed 한정자 대신에 abstract 한정자를 사용할 수도 있지만, sealed 한정자는 외부 파일에서 서브클래스를 만드는 행위 자체를 모두 제한한다.
  * 외모에서 추가적인 서브클래스를 만들 수 없으므로, 타입이 추가되지 않을 거라는 게 보장된다.
* when은 모드를 구분해서 다른 처리를 만들 때 편리하다.

### 태그 클래스와 상태 패턴의 차이
* 상태 패턴은 더 많은 책임을 가진 큰 클래스이다.
* 상태는 변경할 수 있다.
